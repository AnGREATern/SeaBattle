# Техническое задание

## Краткое описание проекта

Проект представляет собой веб-приложение для игры в "Морской бой", где пользователь может сразиться с искусственным интеллектом (ботом). Основная цель - предоставить пользователю возможность увлекательного игрового процесса с тремя уровнями сложности, каждый из которых предлагает уникальную стратегию поведения противника.

Цель проекта: реализовать полноценное игровое приложение "Морской бой" с интерактивным интерфейсом для расстановки кораблей и игрового процесса.


## Описание терминов

### Игра
Экземпляр игровой сессии между пользователем и ботом, включающий этапы расстановки кораблей и боевых действий.

### Игрок
Участник игры. В рамках данного проекта игроками являются пользователь и бот.

### Пользователь
Человек, управляющий своим игровым полем и совершающий ходы через веб-интерфейс.

### Бот
Искусственный интеллект, управляющий своим игровым полем и совершающий ходы в ответ на действия пользователя.

### Матч
Завершенная игра, имеющая результат (победа пользователя или бота).

### Игровое поле
Сетка размером 10x10 клеток, на которой игроки размещают свои корабли и отмечают выстрелы противника.

### Клетка
Минимальная единица игрового поля.

### Состояния клетки (для владельца)
Пустая, Неповрежденный корабль, Поврежденный корабль.

### Состояния клетки (для противника)
Пустая, Промах, Попадание.

### Корабль
Игровой объект, занимающий несколько клеток поля. 

|Тип|Размер
|---|:----:|
|Линкор|4|
|Крейсер|3|
|Эсминец|2|
|Катер|1|

### Ход
Один выстрел, совершаемый игроком. В зависимости от сложности, бот может иметь право на дополнительный ход.

### Выстрел 
Действие игрока по атаке координаты на поле противника.

### Попадание
Выстрел, который попал в клетку, занятую кораблем противника.

### Промах
Выстрел, который попал в пустую клетку.

### Потопление корабля
Состояние, когда все клетки, занимаемые кораблем, поражены.

## Продуктовое описание

Программный продукт представляет собой веб-игру "Морской бой" для одного пользователя, играющего против бота. Игровой процесс состоит из двух обязательных этапов: расстановки кораблей и боевых действий.

### Основной алгоритм:

Начало игры: Пользователь загружает приложение. Ему предлагается начать новую игру с помощью кнопки "Начать новую игру". Затем следует выбор сложности бота.

#### Этап 1: Расстановка кораблей:

Пользователю отображается пустое игровое поле (10x10) и набор кораблей (1 линкор, 2 крейсера, 3 эсминца, 4 катера).

Пользователь должен разместить все корабли на поле, соблюдая правила "Морского боя": корабли не должны соприкасаться друг с другом (даже углами) и должны быть расположены строго горизонтально или вертикально.

Пользователь может выбрать корабль, перетащить его на поле или кликнуть по кораблю, чтобы вращать его.

Интерфейс должен предотвращать некорректную расстановку (пересечения, соприкосновения, выход за пределы поля).

При корректной расстановки кораблей кнопка "Начать" становится кликабельной и пользователь может подтвердить готовность к бою.

#### Этап 2: Боевые действия:

Игровой процесс поочередный, право первого хода определяется случайным образом.

Ход пользователя: Пользователь кликает по клетке на поле противника. Приложение отправляет координаты выстрела на сервер. Сервер определяет результат (попадание/промах) и возвращает его. После этого, в случае промаха, ход переходит к боту, иначе дается возможность сходить повторно.

Ход бота: Бот выбирает клетку на поле пользователя в зависимости от уровня сложности:

1. Простая: Бот делает случайный выстрел по любой еще не атакованной клетке.
2. Средняя: Бот делает случайный выстрел. При попадании начинает атаковать соседние (по горизонтали/вертикали) клетки, пока не потопит корабль, после чего снова переходит к случайным выстрелам.
3. Сложная: Логика "Средней" сложности, но бот ходит два раза подряд.


Игра продолжается до тех пор, пока не будут потоплены все корабли одного из игроков.

Завершение игры: После окончания игры пользователю отображается сообщение о победе или поражении. Переход на начальную страницу.


## Техническое описание

Приложение имеет клиент-серверную архитектуру. Все игровые состояния и логика хранятся и обрабатываются на сервере.

### Сущности

#### Game (Игра):
- `id`: уникальный идентификатор игры;
- `playerBoardId`: идентификатор игрового поля пользователя;
- `botBoardId`: идентификатор игрового поля бота;
- `difficulty`: уровень сложности бота (easy, medium, hard);
- `status`: статус игры (ship_placement, player_turn, bot_turn, player_won, bot_won).

#### Board (Поле):
- `id`: уникальный идентификатор поля;
- `owner`: владелец поля (player, bot);
- `cells`: матрица 10x10 со значениями состояний клеток (empty, ship, miss, damaged, destroyed).
